# Cyber SUWON 2077   

WinAPI 메인 루프 기반의 자체 프레임워크로 구축한 2D 횡스크롤 슈팅 게임  

## 프로젝트 개요

상용 엔진 없이 C++과 DirectX 9, WinAPI만을 사용하여 게임 엔진의 핵심 파이프라인을 직접 설계하고 구현한 프로젝트.
가변 프레임 환경에서의 독립성 확보와 최적화된 충돌 시스템 구축.

개발 환경: C++, WinAPI, DirectX 9  
개발 인원: 1인 개발  
주요 특징: 자체 제작 프레임워크, 데이터 중심 레벨 디자인  

## 주요 구현 특징  
1. 자체 게임 프레임워크 설계 (Update-Render Pipeline)  
* PeekMessage 기반 메인 루프: 윈도우 메시지가 없을 때 즉시 게임 로직을 수행하여 프레임 누락 없는 실시간 연산 보장.  
* 프레임 독립성 확보: deltaTime을 활용하여 CPU 성능에 관계없이 일정한 게임 속도 유지.  
* 싱글톤 패턴: Time, SceneManager 등 핵심 매니저 객체를 싱글톤으로 구현하여 전역적인 접근성 확보 및 리소스 관리 단일화.  
2. 최적화 및 연출 기술  
* 복합 충돌 마스크 (Bit Flag): 비트 연산을 통해 사각/원 충돌 등 여러 충돌체를 한 번에 판별하여, 수많은 투사체가 발생하는 환경에서도 연산 부하 최소화.  
* 지수적 보간 투사체 반사: 단순 반사가 아닌 목표 각도에 부드럽게 수렴하는 지수적 감속 보간을 적용하여 관성이 느껴지는 유려한 곡선 궤적 연출.

3. 스테이지 에디터 및 직렬화  
* 데이터 중심 디자인: 적의 타입, 생성 시간, 이동 경로 등을 txt 데이터로 관리하는 자체 툴 제작.  
* 직렬화 시스템: 코드 수정 없이 데이터 로드만으로 스테이지 구성할 수 있는 환경 구축하여 제작 효율 증대.  

## 트러블슈팅  
✅ 계층적 루프(LateUpdate) 도입을 통한 이미지 떨림 해결  
* 문제: 부모 클래스의 이동 로직과 자식 클래스의 보정 로직 사이의 프레임 동기화 오차로 인해 플레이어 캐릭터가 떨리는 현상 확인.  
* 해결: 프레임워크 수준에서 Update(이동)와 LateUpdate(보정)를 분리하여 이동이 완전히 완료된 후 보정 코드가 실행되도록 구조 개선.

✅ 리소스 로딩 시퀀스 설계를 통한 I/O 병목 해결  
* 문제: 보스 패턴 시작 시 애니메이션 리소스를 실시간으로 로드하면서 프레임이 잠깐 멈추는 현상 발생.  
* 해결: 게임 진입 전 필요한 리소스를 메모리에 미리 적재하는 로딩 시퀀스를 설계하여 런타임 성능 안정성 확보.
